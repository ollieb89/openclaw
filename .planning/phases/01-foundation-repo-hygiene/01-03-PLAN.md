  <action>
**Step 1: Verify detect-secrets is available.**
Run `detect-secrets --version`. If not installed, install via `pip install detect-secrets` or `pipx install detect-secrets`.

**Step 2: Run a full repository scan.**
```bash
cd /home/ollie/Development/Tools/openclaw
detect-secrets scan --baseline .secrets.baseline
```
This updates the baseline with any new findings while preserving existing audit decisions.

**Step 3: Review findings for real secrets.**
```bash
detect-secrets audit .secrets.baseline --report
```
This shows a report of all findings. For each finding:
- If it's a FALSE POSITIVE (test fixtures, example config, schema definitions, placeholder values like `"your-token-here"`): mark as false positive in the baseline
- If it's a REAL SECRET (actual API key, real token, personal info): FLAG IT and note the file + line for remediation

To mark false positives interactively is not supported in non-interactive mode. Instead, use:
```bash
detect-secrets audit .secrets.baseline --json
```
Then parse the output to understand the state.

**Step 4: If real secrets found:**
- For secrets in source files: replace with environment variable references or placeholder values
- For secrets in config examples: replace with clearly fake values (e.g., `"sk-your-key-here"`)
- Do NOT modify git history -- just fix the current state

**Step 5: Re-scan after any fixes:**
```bash
detect-secrets scan --baseline .secrets.baseline
```

**Step 6: Verify the pre-commit hook works.**
Read `.pre-commit-config.yaml` to confirm detect-secrets is configured. Then test:
```bash
pre-commit run detect-secrets --all-files
```
This should pass (exit 0) if the baseline is current and no unbaselined secrets exist.

If `pre-commit` is not installed, verify via:
```bash
cat .pre-commit-config.yaml | grep -A5 detect-secrets
```
Confirm the hook entry exists. The hook itself runs on git commit automatically.

**Important:** The `.secrets.baseline` file may be large (2191 lines). Only modify it via `detect-secrets scan --baseline` commands, never manually edit it.
