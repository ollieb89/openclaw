Create `src/security/audit-log.ts` — the hash-chained audit log writer:

1. Define `AuditLogEntry` type: `{ seq: number; timestamp: string; prevHash: string; event: SecurityEvent; hash: string }`
2. Create `canonicalize(entry: Omit<AuditLogEntry, 'hash'>): string` — deterministic JSON serialization with explicit key ordering. For the `event` field, explicitly order keys: eventType, timestamp, severity, action, then conditionally include sessionKey, channel, detail, meta (only if defined). This prevents hash mismatches from different object construction orders.
3. Create `computeHash(canonicalized: string): string` using `crypto.createHash('sha256').update(canonicalized, 'utf-8').digest('hex')`
4. Create `resolveAuditLogPath(stateDir?: string): string` returning `path.join(resolveStateDir(), 'security', 'audit.jsonl')`. Use `resolveStateDir` from `src/config/paths.ts`.
5. Implement singleton writer with promise-chain serialization (follow pattern from `src/cron/run-log.ts`):
   - Module-level `let writeChain = Promise.resolve()`, `let lastHash = 'GENESIS'`, `let lastSeq = 0`, `let initialized = false`
   - `appendAuditEntry(event: SecurityEvent): void` — synchronously queues onto writeChain (fire-and-forget). The chained promise: (a) if not initialized, call `recoverState()` to read last valid line from existing log; (b) build entry with seq=lastSeq+1, timestamp from event, prevHash=lastHash; (c) compute hash; (d) ensure directory exists with `0o700`; (e) append JSON line with `fs.promises.appendFile` and ensure file permissions `0o600`; (f) update lastHash and lastSeq.
   - `recoverState()`: read last line of audit.jsonl (if exists), parse it, set lastHash and lastSeq. If last line is truncated JSON, skip it and try second-to-last. If file doesn't exist, keep GENESIS defaults.
   - Export `resetAuditWriter()` for testing (resets module state to defaults).

Create `src/security/audit-log-verify.ts` — chain verification:

1. Export `VerifyResult = { valid: true; entryCount: number } | { valid: false; entryCount: number; failedAtSeq: number; error: string }`
2. Export `verifyAuditLogChain(logPath: string): Promise<VerifyResult>` — uses `node:readline` createInterface over createReadStream for memory-efficient streaming. For each line: parse JSON, check seq continuity (must equal entryCount), check prevHash matches previous entry's hash (or GENESIS for first), recompute hash via canonicalize+computeHash and compare. Return failure details on first mismatch. Handle truncated trailing line as warning (not failure) — if JSON.parse fails on last line only, skip it and return valid with the count of successfully verified entries.
3. Import `canonicalize` and `computeHash` from `audit-log.ts` (shared between writer and verifier to guarantee consistency).

Create tests in `src/security/audit-log.test.ts`:
- Genesis entry: first append creates entry with seq=1, prevHash="GENESIS"
- Chaining: second append has prevHash equal to first entry's hash
- Concurrent writes: call appendAuditEntry 5 times synchronously, verify all 5 entries have sequential seq numbers and valid chain
- Crash recovery: write 3 entries, reset writer state, write 4th entry — verify it chains correctly from entry 3
- Use a temp directory for all tests (vi.stubEnv or mock resolveStateDir)

Create tests in `src/security/audit-log-verify.test.ts`:
- Valid chain: write 3 entries, verify returns valid with entryCount=3
- Tampered entry: write 3 entries, modify middle entry's detail field in file, verify returns invalid with failedAtSeq=2
- Deleted entry: write 3 entries, remove middle line, verify returns invalid (sequence gap)
- Inserted entry: write 3 entries, duplicate middle line, verify returns invalid (prevHash mismatch)
- Empty file: verify returns valid with entryCount=0
- Truncated last line: write 3 entries, append partial JSON, verify returns valid with entryCount=3

Use Vitest. Colocate tests next to source. Use tmp directories to avoid polluting real state.
