Add a non-blocking audit log verification call early in `startGatewaySidecars()`:

1. Import `verifyAuditLogChain` from `../security/audit-log-verify.js`, `resolveAuditLogPath` from `../security/audit-log.js`, and `emitSecurityEvent` from `../security/event-logger.js` (if not already imported).
2. Import `existsSync` from `node:fs` if not already available.
3. At the very beginning of `startGatewaySidecars()` (before the browser control server start), add the audit log verification as a non-blocking call:

```typescript
// Verify security audit log integrity (non-blocking)
const auditLogPath = resolveAuditLogPath();
if (existsSync(auditLogPath)) {
  verifyAuditLogChain(auditLogPath)
    .then((result) => {
      if (!result.valid) {
        params.log.warn(
          `Security audit log tamper detected at entry ${result.failedAtSeq}: ${result.error}`,
        );
        emitSecurityEvent({
          eventType: "policy.violation",
          timestamp: new Date().toISOString(),
          severity: "critical",
          action: "detected",
          detail: `Audit log integrity check failed at entry ${result.failedAtSeq}: ${result.error}`,
        });
      }
    })
    .catch((err) => {
      params.log.warn(`Security audit log verification failed: ${String(err)}`);
    });
}
```

Key requirements:
- Non-blocking: use `.then()` pattern, do NOT await. Gateway startup must not be delayed by verification.
- If log file doesn't exist yet, skip verification silently (no log file = no events yet = nothing to verify).
- On tamper detection: log warning AND emit a security event (policy.violation, critical severity).
- On verification error (e.g., read failure): log warning, do not emit security event.
- Do NOT prevent gateway startup on tamper detection â€” warn only.
